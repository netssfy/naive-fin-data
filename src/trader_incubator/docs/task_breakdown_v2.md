# 交易员孵化系统任务拆解（基于 requirement_v2.md）

## 拆解原则
- 每个任务都满足：有明确产物、可独立验收、规模可控（建议 0.5~2 天）。
- 每个任务包含：目标、交付物、验证方式、完成定义（DoD）。
- 任务编号采用 `Txx`，可直接进入迭代看板。

## 任务清单

### T01 创建赛季数据模型
- 目标：落地 Season 数据结构与状态机字段。
- 交付物：
  - `Season` 实体/模型定义。
  - 状态枚举：`DRAFT/RUNNING/PAUSED/ENDED`。
- 验证方式：
  - 单元测试：字段必填和枚举合法性校验。
- DoD：
  - 模型可被创建与序列化，测试通过。

### T02 赛季创建接口
- 目标：提供创建赛季能力。
- 交付物：
  - `POST /seasons` 接口。
  - 参数校验与错误码约定。
- 验证方式：
  - 接口测试：正常创建、缺失市场、非法时间三类用例。
- DoD：
  - 返回 `season_id`，错误场景返回可读信息。

### T03 赛季查询接口
- 目标：支持查看赛季配置和状态。
- 交付物：
  - `GET /seasons/{season_id}` 接口。
- 验证方式：
  - 接口测试：存在/不存在赛季。
- DoD：
  - 返回字段完整，404 场景明确。

### T04 代码池约束校验器
- 目标：实现“仅允许代码池内交易”的独立校验模块。
- 交付物：
  - `ticker_pool_validator` 组件。
- 验证方式：
  - 单测覆盖：有池命中、无池放行、越界拒绝。
- DoD：
  - 拒绝时返回标准错误原因。

### T05 交易员数据模型
- 目标：定义 Trader、风格、策略版本、风控参数结构。
- 交付物：
  - `Trader` 实体/模型及枚举状态。
- 验证方式：
  - 单元测试：序列化、默认值、状态切换。
- DoD：
  - 可持久化并可关联 `season_id`。

### T06 对接 `帅帅` 生成器入口
- 目标：提供统一调用入口创建交易员。
- 交付物：
  - `generate_traders(season_config, n)` 服务接口。
- 验证方式：
  - Mock 测试：返回 N 个交易员初始配置。
- DoD：
  - 失败路径有错误日志且可重试。

### T07 风格相似度计算模块
- 目标：实现交易员风格相似度评估。
- 交付物：
  - `style_similarity(a, b)` 方法与报告结构。
- 验证方式：
  - 单测：边界值（0%、50%、>50%）判断正确。
- DoD：
  - 两两校验可输出合规/不合规清单。

### T08 赛季启动流程编排
- 目标：赛季启动时自动生成交易员并做相似度校验。
- 交付物：
  - `start_season` 编排流程。
- 验证方式：
  - 集成测试：启动成功、相似度不合规时阻断。
- DoD：
  - 生成结果与校验报告可查询。

### T09 市场日历与交易窗口判断
- 目标：实现“当前是否交易时间”的基础判断。
- 交付物：
  - `is_market_open(market, timestamp)` 组件。
- 验证方式：
  - 单测：开市、闭市、周末/节假日场景。
- DoD：
  - 返回值可被调度器直接消费。

### T10 分钟级调度器骨架
- 目标：实现每分钟触发的主循环框架。
- 交付物：
  - Scheduler 服务与 `run_id` 生成。
- 验证方式：
  - 集成测试：连续触发 3 轮，产生 3 个唯一 `run_id`。
- DoD：
  - 调度日志完整，失败不崩溃。

### T11 行情拉取适配器（仿真源）
- 目标：提供统一行情读取接口。
- 交付物：
  - `MarketDataProvider` 抽象和默认实现。
- 验证方式：
  - 单测：正常返回、超时异常、重试逻辑。
- DoD：
  - 调度器可成功获取分钟行情快照。

### T12 信号生成调用链
- 目标：从交易员获取 BUY/SELL/HOLD 信号。
- 交付物：
  - `generate_signal(trader, market_snapshot)` 入口。
- 验证方式：
  - Mock 集成测试：多交易员并发生成信号。
- DoD：
  - 失败交易员隔离，不影响其他交易员。

### T13 风控规则引擎（首版）
- 目标：对信号进行仓位/标的/价格等校验。
- 交付物：
  - 风控校验模块与拒单原因枚举。
- 验证方式：
  - 单测：超仓位、非代码池、异常价格场景。
- DoD：
  - 每次拒单都有结构化原因。

### T14 仿真下单与成交模块
- 目标：将通过风控的信号转换为订单并仿真成交。
- 交付物：
  - `Order`、`Fill` 处理逻辑。
- 验证方式：
  - 集成测试：BUY/SELL 路径、部分拒单路径。
- DoD：
  - 订单状态流转正确（CREATED/FILLED/REJECTED）。

### T15 持仓与账本更新
- 目标：成交后更新持仓、现金、净值。
- 交付物：
  - `PositionLedger` 模块。
- 验证方式：
  - 单测：买入增仓、卖出减仓、净值重算。
- DoD：
  - 任意成交后账本可重算且一致。

### T16 交易记录落库
- 目标：保存信号、订单、成交、账本快照。
- 交付物：
  - Repository/DAO 与表结构迁移脚本。
- 验证方式：
  - 集成测试：一次交易循环后数据完整可查。
- DoD：
  - 可通过 `run_id` 串联全链路数据。

### T17 策略锁定机制（交易时段）
- 目标：交易时段禁止策略版本修改。
- 交付物：
  - 策略发布前置校验器。
- 验证方式：
  - 单测：交易时段发布失败，非交易时段允许。
- DoD：
  - 拒绝信息明确且可追踪。

### T18 非交易时段复盘任务
- 目标：每日生成交易员复盘摘要。
- 交付物：
  - `daily_review_job` 与复盘存储结构。
- 验证方式：
  - 任务测试：每个交易员生成一条复盘记录。
- DoD：
  - 复盘包含收益、回撤、关键交易说明。

### T19 市场信息汇总任务
- 目标：在非交易时段采集并汇总市场信息。
- 交付物：
  - `market_digest_job` 输出结构化摘要。
- 验证方式：
  - 任务测试：产出摘要并关联日期与市场。
- DoD：
  - 可被策略建议任务直接读取。

### T20 策略调整草案任务
- 目标：根据复盘与市场摘要产出策略建议。
- 交付物：
  - `strategy_patch_draft` 数据结构和生成流程。
- 验证方式：
  - 任务测试：草案状态默认 `PENDING_REVIEW`。
- DoD：
  - 不自动生效，需后续审批/发布。

### T21 收益指标计算器
- 目标：实现累计收益、日收益、最大回撤、胜率计算。
- 交付物：
  - `performance_metrics` 模块。
- 验证方式：
  - 单测：使用固定样本校验指标值。
- DoD：
  - 输出可直接用于排行榜接口。

### T22 收益天梯接口
- 目标：提供按赛季排序的排行榜 API。
- 交付物：
  - `GET /seasons/{season_id}/leaderboard`
- 验证方式：
  - 接口测试：排序稳定、同分规则一致。
- DoD：
  - 刷新延迟满足 1 分钟内。

### T23 Web 赛季概览页
- 目标：展示赛季状态、市场、时间、交易员数量。
- 交付物：
  - 页面与数据请求逻辑。
- 验证方式：
  - UI 测试：页面加载成功，字段显示完整。
- DoD：
  - 可切换不同赛季查看。

### T24 Web 交易员列表与详情页
- 目标：展示交易员风格、策略版本、当前绩效。
- 交付物：
  - 列表页 + 详情页。
- 验证方式：
  - UI 测试：筛选、跳转、详情数据正确。
- DoD：
  - 风格信息与策略版本可见。

### T25 Web 交易记录页
- 目标：展示信号、订单、成交，并支持筛选。
- 交付物：
  - 交易记录表格页。
- 验证方式：
  - UI 测试：按交易员/时间/标的筛选生效。
- DoD：
  - 行为链路字段完整可追溯。

### T26 Web 收益天梯页
- 目标：展示收益排名与关键指标。
- 交付物：
  - 排行榜页及刷新机制。
- 验证方式：
  - UI 测试：1 分钟内刷新，排名显示正确。
- DoD：
  - 指标单位、时间戳展示清晰。

### T27 结构化日志规范
- 目标：统一关键流程日志字段。
- 交付物：
  - 日志规范文档 + logger 封装。
- 验证方式：
  - 检查日志包含 `run_id/season_id/trader_id`。
- DoD：
  - 故障排查可通过字段快速定位。

### T28 错误告警与重试策略
- 目标：为行情失败、落库失败、任务失败建立重试与告警。
- 交付物：
  - 重试策略配置与告警钩子。
- 验证方式：
  - 故障注入测试：验证重试次数与告警触发。
- DoD：
  - 不因单点失败导致系统整体停摆。

### T29 端到端验收脚本
- 目标：一键跑通“创建赛季 -> 交易循环 -> 排行展示”。
- 交付物：
  - E2E 脚本与示例数据。
- 验证方式：
  - CI 或本地运行成功并产出报告。
- DoD：
  - 关键路径无人工介入可跑通。

### T30 文档与运行手册
- 目标：补齐开发和运维最小文档。
- 交付物：
  - README（启动步骤、配置项、常见问题）。
- 验证方式：
  - 新成员按文档可在 30 分钟内跑起系统。
- DoD：
  - 文档经过一次新人试跑验证。

## 建议执行顺序（可直接按阶段建迭代）
1. 基础域模型与接口：T01-T05
2. 交易员生成与赛季启动：T06-T08
3. 交易主链路：T09-T16
4. 非交易任务：T17-T20
5. 指标与排行榜：T21-T22
6. Web 展示：T23-T26
7. 稳定性与交付完善：T27-T30

## 每任务验收模板（可复用）
- 输入：任务前置数据/依赖
- 操作：执行步骤（接口调用/任务触发/UI 操作）
- 预期结果：可观察输出
- 证据：测试报告、日志片段、截图、数据记录 ID
