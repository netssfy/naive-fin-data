# 交易员孵化系统

## 系统主要角色

### 世界系统
* 整个交易员孵化系统的运行时
* 应该基于pi-agent，deer-flow这类AI Coding Harness来实现
* 整个系统是混合实现的，由AI Coding Harness驱动，部分生成的代码则使用cli的形式运行

### 交易所
* 由python实现的交易执行框架和引擎
* 加载交易员的交易策略
* 每分钟触发一次，提供最新的数据给交易员，并执行交易员的策略
* 记录交易员的交易行为

### 帅帅
* 创造交易员
* 自身是一个skill，可以被AI Coding Harness来执行
* 创造的交易员主体也是一个skill，并在skill中赋予交易员一些特性

### 交易员
* 交易员有一个名字
* 交易员有自己的风格
* 在交易时间段
    * 会根据自己的交易策略，风控策略来进行交易
    * 会发出可交易信号
    * 会把每天的交易记录下来
* 在非交易时间段
    * 会复盘交易
    * 获取最新的市场信息，并进行研究
    * 调整交易策略

### 交易赛季
一个交易赛季可以容纳多名交易员在赛季中进行交易比赛，动态更新收益率天梯

* 一个赛季必须有开始时间
* 一个赛季可以有结束时间，也可以是无结束时间的永恒赛季
* 一个赛季可以容纳多名交易员
* 一个赛季必须指定交易的市场，比如A股，港股，美股
* 一个赛季可以指定可交易的股票代码池里，所有交易员的交易标的物必须都在代码池里，不得交易非代码池里的股票。如果未指定股票代码池，则表示市场内所有股票皆可交易
* 赛季开始时，会用`帅帅`来创造交易员，并赋予不同的风格
* `帅帅`必须构建风格不同的交易员，两两之间的相似度不能超过50%

### 可视化面板
* 一个现代化的web页面，显示赛季，交易员，交易记录等信息

## 主要工作流

### 初始化阶段
1. 用户创建一个赛季，指定赛季信息
2. `帅帅`会根据赛季指定的信息，创建出对应的交易员
3. 交易员初始化，根据历史数据以及自身风格实现第一版的交易代码

### 交易阶段
1. 在交易时间内，系统应该启动`交易所`来执行交易员的策略
2. 在交易时间内，交易员不应该再修改交易策略
